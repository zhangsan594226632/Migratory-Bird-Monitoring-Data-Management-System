## 一、项目概述

### 1\.1 项目背景

随着生态保护、鸟类观测行业快速发展，野生动植物保护机构、观鸟爱好者、生态科研人员需要统一线上平台记录、查阅候鸟观测数据、迁徙路线、行业新闻，同时搭建交流社区分享观测心得。传统线下 Excel、零散文档记录候鸟观测数据存在数据易丢失、无法共享、迁徙路线无法集中展示、观测人员缺少交流渠道、数据无可视化统计分析等痛点。
本系统采用 SpringBoot\+Vue 前后端分离架构，搭建**候鸟监测数据管理系统**，划分三类角色：普通游客 / 观鸟用户、录入员、系统管理员；分为前台展示交互端与后台数据管理端，完整覆盖候鸟数据管理、迁徙路线管理、新闻资讯、论坛交流、用户管理、可视化数据统计全业务流程，实现候鸟观测数据数字化、可视化、协同化管理，助力生态鸟类监测工作。

### 1\.2 开发技术栈

#### 后端（SpringBoot）

1. 核心框架：SpringBoot 2\.7\.x，标准化 RESTful 接口，分层架构解耦开发

2. 持久层：MyBatis\-Plus，实现分页、多条件模糊检索、批量增删改查

3. 数据库：MySQL 8\.0，存储用户、候鸟种类、候鸟观测数据、迁徙路线、新闻资讯、论坛帖子、评论、轮播图全量业务数据

4. 权限鉴权：JWT Token 无状态登录认证，区分普通用户、录入员、管理员三级角色，拦截未登录、越权访问接口

5. 安全机制：用户密码 MD5 加密存储，表单参数双重校验，登录会话时效控制

6. 文件上传：本地存储方案，支持候鸟实拍图、迁徙路线配图、资讯封面、用户头像上传预览

7. 数据可视化：ECharts 后端统计接口，输出饼图、柱状图数据，实现候鸟种类分布、观测数量统计

8. 工具类：Hutool，封装日期处理、加密、文件操作、分页工具

#### 前端（Vue3 \+ Element Plus）

1. 核心框架：Vue3 \+ VueRouter \+ Pinia 全局状态管理

2. UI 组件库：Element Plus（管理员 / 录入员后台）、轻量化简约前台用户页面

3. 网络请求：Axios 全局请求 / 响应拦截器，自动携带 Token、捕获登录失效、统一异常弹窗提示

4. 可视化图表：ECharts 渲染候鸟种类分析饼图、候鸟观测统计柱状图

5. 通用组件：首页轮播图、图片上传组件、卡片列表、分页控件、搜索筛选、详情弹窗、评论列表

### 1\.3 系统角色划分

1. **普通观鸟用户（前台游客）**
账号注册登录、浏览首页轮播、查看候鸟观测数据、迁徙路线、生态新闻资讯；论坛发帖、评论交流；收藏感兴趣的候鸟数据；个人中心维护资料、查看我的收藏、我的帖子。

2. **录入员**
登录后台，负责录入、维护候鸟观测数据、迁徙路线信息，发布生态新闻，管理自身发布的论坛内容，无用户管理、系统配置权限。

3. **系统管理员**
最高权限后台管控：管理全部用户、录入员账号；维护候鸟种类字典；统一管理候鸟数据、迁徙路线、新闻资讯、论坛帖子；首页轮播图配置；系统数据可视化统计查看；删除违规帖子、评论。

### 1\.4 整体业务流程

1. 注册登录流程：普通用户前台完成账号注册；后台区分管理员 / 录入员独立登录入口；校验账号密码签发 Token，按角色跳转对应页面。

2. 普通用户前台流程：浏览首页→查看候鸟数据 / 迁徙路线 / 新闻资讯→论坛发帖互动→收藏候鸟观测数据→个人中心管理收藏与帖子。

3. 录入员后台流程：登录后台录入、修改候鸟观测数据、迁徙路线、发布新闻、管理自己发布的论坛内容。

4. 管理员后台流程：维护基础数据（候鸟种类）、管理所有用户 / 录入员、全量管控候鸟数据、迁徙路线、新闻、论坛；配置首页轮播图；查看候鸟观测数据可视化统计报表。

### 2\.1 登录注册模块

#### 2\.1\.1 前台用户登录页面

简洁登录表单，包含账号、密码输入框、记住密码勾选框、登录按钮，底部跳转注册页面；未登录状态访问详情、论坛、收藏功能会弹窗提示先登录。

#### 2\.1\.2 前台用户注册页面

必填项：用户账号、姓名、密码、确认密码；选填：头像上传、性别、手机号码；前端校验非空、密码一致性，后端校验账号唯一性，完成注册入库。

#### 2\.1\.3 后台登录页面

分双标签切换：管理员、录入员登录入口，独立账号体系，权限隔离，登录后进入对应管理后台首页。

### 2\.2 前台用户功能模块（顶部导航：首页、论坛交流、候鸟数据、迁徙路线、个人中心）

#### 2\.2\.1 系统首页

1. 顶部全屏轮播横幅展示生态风景宣传图；

2. 候鸟数据展示区域：卡片分页展示候鸟观测记录，卡片包含实拍图，点击查看更多跳转完整列表；

3. 迁徙路线展示区域：卡片展示各地候鸟迁徙路线配图与路线名称；

4. 新闻资讯区域：展示生态、鸟类保护相关新闻卡片，支持查看更多资讯。

#### 2\.2\.2 候鸟数据模块

1. 分页卡片展示全部候鸟观测数据，支持名称模糊检索；

2. 点击卡片进入候鸟数据详情页：展示候鸟实拍图、候鸟种类、观测日期、观测地点、观测数量完整信息；

3. 收藏功能：详情页支持收藏 / 取消收藏该条候鸟观测数据。

#### 2\.2\.3 迁徙路线模块

卡片式展示全部候鸟迁徙路线，每条路线附带实拍风景图、路线名称；点击查看路线完整详情，了解候鸟迁徙途经地点、季节规律。

#### 2\.2\.4 新闻资讯模块

展示生态保护、鸟类科普、观测活动相关新闻，支持标题检索；点击查看资讯全文，图文结合展示科普内容。

#### 2\.2\.5 论坛交流模块

1. 顶部搜索框按帖子标题、内容检索；

2. 分页展示全部用户发布帖子，显示帖子标题、发布人；

3. 新增发帖：登录用户可发布新帖子，填写标题、正文分享观鸟心得；

4. 帖子详情：查看正文、所有用户评论，支持发表评论互动。

#### 2\.2\.6 个人中心

用户修改个人资料、更换头像、修改登录密码；查看我的收藏（收藏的候鸟数据）、我发布的全部论坛帖子。

### 2\.3 后台管理功能模块（左侧侧边导航菜单）

#### 2\.3\.1 后台首页（数据统计看板）

1. 数据卡片展示候鸟数据总条数；

2. ECharts 可视化图表：

    - 候鸟种类分析饼图：统计各类候鸟观测记录占比；

    - 候鸟观察统计柱状图：展示每条候鸟数据观测数量对比，直观呈现热门观测候鸟。

#### 2\.3\.2 候鸟信息管理

1. **候鸟种类管理**：维护候鸟分类字典，新增、修改、删除候鸟种类名称，用于前台数据归类筛选；

2. **候鸟数据管理**：全量管理观测记录，支持按名称检索、新增观测数据、修改观测信息、删除无效记录，每条数据包含种类、实拍图、观测时间、地点、数量。

#### 2\.3\.3 用户管理

管理所有前台注册普通用户账号，支持用户名检索、查看用户详情、修改用户信息、封禁违规用户。

#### 2\.3\.4 迁徙路线管理

统一维护候鸟迁徙路线数据，新增路线配图、路线名称、途经地点、迁徙季节描述；支持修改、删除、检索路线信息。

#### 2\.3\.5 后台管理子模块

1. **轮播图管理**：维护首页顶部轮播宣传图，上传图片、设置排序、启用 / 关闭展示；

2. **新闻资讯管理**：后台表格展示全部新闻，按标题检索，支持新增资讯、修改图文内容、删除过期资讯；

3. **录入员管理（仅管理员可见）**：新增录入员账号、重置密码、删除录入员，管控后台数据录入权限人员。

#### 2\.3\.6 论坛管理

1. **我的发布（录入员 / 管理员）**：查看自身发布的论坛帖子，新增、修改、删除帖子；

2. **全量论坛管理（仅管理员）**：查看平台所有用户帖子，检索帖子、查看帖子下全部评论、删除违规帖子与不良评论，维护社区交流秩序。

### 3\.10 banner 首页轮播图表

id、img\_url 轮播图片、sort 排序、is\_show 是否前台展示

## 四、核心业务与安全流程

### 4\.1 三级权限鉴权流程

1. 前台普通用户登录：仅可访问前台页面，仅能操作自己的收藏、帖子、评论；无后台访问权限。

2. 录入员后台登录：仅可录入 / 修改候鸟数据、迁徙路线、新闻、自身帖子；无法管理用户、录入员、删除他人帖子。

3. 管理员后台登录：拥有全平台数据增删改查、用户管控、系统配置、违规内容删除权限。

4. 后端拦截器校验 Token 内角色标识，无对应权限接口直接返回 403 禁止访问。

### 4\.2 候鸟观测数据完整业务流程

1. 录入员后台新增候鸟观测记录，上传实拍图、填写种类、观测时间地点数量；

2. 数据存入 bird\_data 表，前台首页、候鸟数据列表自动加载展示；

3. 普通用户浏览详情，可收藏该条观测数据，存入收藏表；

4. 管理员后台可查看全部数据，通过 ECharts 生成种类分布、观测数量统计图表；

5. 数据过时 / 错误时，录入员、管理员可修改或删除记录。

### 4\.3 论坛交流闭环流程

1. 登录用户前台发布新帖子，填写标题正文提交；

2. 所有登录用户可查看帖子、发表评论；

3. 发布用户可修改、删除自己的帖子；

4. 管理员后台审核全量帖子，删除违规内容与不良评论，维护社区环境。

## 五、项目部署说明

### 5\.1 SpringBoot 后端部署步骤

1. 执行项目 SQL 脚本初始化 MySQL 数据库，预置管理员、录入员测试账号；

2. 修改`application.yml`配置：

    - MySQL 数据库连接地址、账号、密码；

    - JWT 加密密钥、Token 过期时长；

    - 文件上传本地存储路径、文件大小限制；

    - 后端服务启动端口（默认 8080）；

3. Maven 打包：`mvn clean package -DskipTests`生成 jar 包；

4. 服务器启动命令：`java -jar bird-monitor.jar`。

### 5\.2 Vue 前端部署步骤

1. 前端目录执行`npm install`安装全部依赖；

2. 修改 axios 请求基地址，指向后端服务接口；

3. 打包生产静态资源：`npm run build`输出 dist 文件夹；

4. Nginx 托管 dist 静态文件，配置 /api 反向代理转发至后端 8080 端口；

5. 区分前台访问路由、管理员 / 录入员后台路由，独立访问入口。

## 六、系统特色与创新亮点

### 6\.1 业务功能亮点

1. **生态观测数据全生命周期管理**：从候鸟数据录入、前台展示、用户收藏、后台统计形成完整闭环，完整留存鸟类观测原始记录，便于生态科研追溯；

2. **迁徙路线可视化展示**：图文结合展示候鸟迁徙路线、途经地域、迁徙季节，直观科普鸟类迁徙规律；

3. **分层社区论坛交流**：观鸟爱好者线上分享观测经验，配套评论互动功能，搭建生态爱好者交流平台；

4. **多维度数据可视化统计**：饼图、柱状图直观分析候鸟种类分布、观测数量，为生态保护工作提供数据支撑；

5. **三级精细化角色权限**：游客、录入员、管理员权限完全隔离，数据录入、审核、管控分工清晰，适配真实生态机构工作流程。

### 6\.2 系统安全亮点

1. 密码不可逆 MD5 加密存储，杜绝明文泄露；

2. JWT 无状态登录，Token 设置过期时间，退出登录销毁会话，防止账号劫持；

3. 三层参数校验：前端表单校验 \+ 后端接口参数校验 \+ 数据库约束，拦截非法脏数据；

4. 接口角色权限拦截，不同账号只能操作自身权限内数据，越权访问直接拦截；

5. 所有图片、文件上传做格式、大小校验，防止恶意脚本文件上传攻击。

### 6\.3 拓展优势

1. 前后端完全分离，低耦合分层架构，便于后续迭代新增 GPS 迁徙轨迹地图、观测任务打卡、鸟类百科知识库、观测报告导出 Excel 功能；

2. 基础数据字典化管理：候鸟种类、迁徙分类后台可视化维护，无需修改代码即可新增分类；

3. 轻量化前台页面，适配电脑、平板多端访问，操作简单，降低观鸟爱好者使用门槛。

## 七、项目总结

本《基于 SpringBoot\+Vue 候鸟监测数据管理系统》面向生态鸟类观测、野生动植物保护场景开发，采用前后端分离架构，解决传统线下记录候鸟观测数据分散、无交流渠道、缺少数据统计分析的痛点。系统划分普通用户、录入员、管理员三类角色，实现候鸟观测数据、迁徙路线、生态新闻、论坛社区一体化线上管理；配套 ECharts 可视化数据统计、三级权限安全管控、文件图片上传等完整配套功能。
项目代码结构规范、业务贴合生态观测真实工作场景，既满足计算机专业毕业设计、课程设计交付要求，也可优化迭代落地到鸟类保护机构、湿地公园、观鸟社团实际业务使用，具备良好实用性、可维护性与拓展开发空间。
